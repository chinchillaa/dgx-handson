2
AIを「強く」する
SFT・RLHF・RAG — 目的に応じた強化手法の選択
約 2–3h
トピック
強化の選択肢マップ：SFT / RLHF(PPO) / DPO / RAG / Prompt Engineering
SFT：データセット設計・過学習・フォーマット
RLHF / DPO：人間フィードバックの組み込み方
RAG：Retrieval設計・チャンク戦略・Embeddingモデル選択
量子化・LoRA：リソース制約下での効率化
評価：BERTScore / ROUGE / LLM-as-judge
ハンズオン
実装LLaMA 3をLoRA + SFTTrainerで日本語タスクにSFT（DGX上で実行）
実装DPO用のpreference datasetを作成・学習、SFTとの出力差を比較
実装LangChain + ChromaDBでRAGパイプライン構築、チャンクサイズ比較実験
課題SFT前後・RAG有無で同一質問に回答させ、評価指標で定量比較
3
AIエージェントを作ろう
Tool use・メモリ・マルチエージェントで自律動作を実現
約 2–3h
トピック
エージェントとは何か：ReAct / Plan-and-Execute / Reflexion
Tool use：function calling・MCP・APIの抽象化
メモリ：Short-term / Long-term / Episodic
マルチエージェント：オーケストレーションと役割分担
信頼性・安全性：ハルシネーション対策・ガードレール
本番運用：観測性・ログ・コスト管理
ハンズオン
実装LangGraph + 検索ツールでReActエージェントを構築（Python）
実装外部API（天気・DB）をtool登録、エージェントが自律呼び出し
実装2エージェント構成（Planner + Executor）でレポート自動生成タスク
課題意図的なバグのあるtool定義を渡し、エラーリカバリの挙動を観察・修正
