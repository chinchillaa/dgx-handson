Section 1

線形回帰：最もシンプルな「予測関数」

線形回帰は、入力 $x$ から出力 $y$ を予測する 最もシンプルな関数 です。 「線形（linear）」とは、入力と出力の関係が 直線 で表せることを意味します。

例として、「部屋の広さ（m²）から家賃（万円）を予測する」問題を考えます。 データを散布図に描いたとき、おおよそ直線に乗っているなら線形回帰が使えます。

モデルの定義

$$ \hat{y} = wx + b $$
$x$入力（例：部屋の広さ）
$\hat{y}$予測値（例：予測家賃）
$w$重み（weight）— パラメータ①
$b$バイアス（bias）— パラメータ②

学習とは $w$ と $b$ の値を「データに合うよう」自動で決める 作業です。 モデルの構造（直線）は人間が決め、パラメータの値はデータから決まります。

コード例 — データ生成と初期予測

import numpy as np
import matplotlib.pyplot as plt

# 実行時間: 数秒

# ── データ生成 ──────────────────────────────────────
# 真のモデル: y = 2x + 1 にノイズを乗せる
np.random.seed(42)
x = np.linspace(0, 10, 50)
y_true = 2 * x + 1
y = y_true + np.random.randn(50) * 2   # 観測値（ノイズあり）

# ── 初期パラメータでの予測 ──────────────────────────
w, b = 0.0, 0.0   # 初期値はゼロ
y_pred = w * x + b

print(f"初期パラメータ: w={w}, b={b}")
print(f"予測値の例: {y_pred[:5]}")   # すべて 0.0 — まったく予測できていない
出力:
初期パラメータ: w=0.0, b=0.0
予測値の例: [0. 0. 0. 0. 0.]

パラメータが $w=0, b=0$ では何も予測できていません。 次のセクションで「どれだけ外れているか」を計測します。

Section 2

損失関数：「外れ具合」を数値にする

予測値 $\hat{y}$ と正解値 $y$ の「ずれ」を1つの数値にまとめたものが 損失関数（loss function） です。 損失が大きい → 予測が外れている、損失が小さい → 予測が当たっている、という指標です。

回帰問題でよく使われるのは 平均二乗誤差（MSE: Mean Squared Error） です。 「ずれを二乗して平均をとる」だけのシンプルな計算です。

なぜ「二乗」するのか？

  • ずれにはプラス・マイナスがある。単純に合計するとキャンセルしてしまう
  • 二乗するとすべてプラスになり、「大きなずれ」をより強くペナルティにできる
  • 二乗は微分しやすく、後の勾配降下法と相性が良い

MSE の定義

$$ L(w, b) = \frac{1}{N} \sum_{i=1}^{N} \left( y_i - \hat{y}_i \right)^2 = \frac{1}{N} \sum_{i=1}^{N} \left( y_i - (wx_i + b) \right)^2 $$
$N$データ点の総数
$y_i$$i$ 番目の正解値
$\hat{y}_i$$i$ 番目の予測値 $= wx_i + b$
$L(w,b)$損失（これを小さくすることが「学習」の目的）

コード例 — MSE の計算

def mse_loss(y_true, y_pred):
    """平均二乗誤差（MSE）を計算する"""
    return np.mean((y_true - y_pred) ** 2)

# 初期パラメータ（w=0, b=0）での損失
loss_init = mse_loss(y, y_pred)
print(f"初期損失 (w=0, b=0):   {loss_init:.2f}")

# 真のパラメータ（w=2, b=1）での損失
y_pred_true = 2 * x + 1
loss_true = mse_loss(y, y_pred_true)
print(f"真のパラメータの損失:    {loss_true:.2f}")   # ノイズ分だけ残る
出力:
初期損失 (w=0, b=0): 109.45
真のパラメータの損失: 3.87

真のパラメータ近くでは損失が格段に小さくなります。 「損失を最小化するパラメータを探す」ことが学習のゴールです。

概念：損失の「地形」

$w$ と $b$ の値を変えると損失 $L$ が変わります。 これを3次元で描くと お椀型の曲面（下図）になります。 底（最小値）が「最もデータに合ったパラメータ」の位置です。

学習 = 現在地から底（損失最小）に向かって少しずつ動く

Section 3

勾配降下法：「坂を下る」ようにパラメータを更新する

損失関数のお椀の底を求めるには、「今いる場所の傾き（勾配）を計算して、 坂を下る方向に少し移動する」 を繰り返します。 この手続きが 勾配降下法（Gradient Descent） です。

「どのくらいの歩幅で移動するか」を決める定数が 学習率（learning rate） $\alpha$ です。 大きすぎると底を飛び越え、小さすぎると収束が遅くなります。

勾配の計算（MSE の偏微分）

損失 $L$ が $w$ と $b$ それぞれに対してどのくらい変化するかを求めます。 この「変化の割合」が 勾配 です。

$$ \frac{\partial L}{\partial w} = -\frac{2}{N} \sum_{i=1}^{N} x_i \left( y_i - \hat{y}_i \right) $$ $$ \frac{\partial L}{\partial b} = -\frac{2}{N} \sum_{i=1}^{N} \left( y_i - \hat{y}_i \right) $$

導出は気にしなくて大丈夫です。「誤差 $(y_i - \hat{y}_i)$ が大きいほど勾配も大きい」という 直感だけ押さえてください。

パラメータ更新則

勾配の 反対方向（損失が下がる方向）に学習率 $\alpha$ だけ移動します。

$$ w \leftarrow w - \alpha \frac{\partial L}{\partial w} $$ $$ b \leftarrow b - \alpha \frac{\partial L}{\partial b} $$
💡 これを何百・何千回繰り返すことで、パラメータが少しずつ「正解」に近づいていきます。 これが「学習」の正体です。

コード例 — NumPy で勾配降下法を手実装

import numpy as np
import matplotlib.pyplot as plt

# 実行時間: 約5秒

# ── データ生成 ──────────────────────────────────────
np.random.seed(42)
x = np.linspace(0, 10, 50)
y = 2 * x + 1 + np.random.randn(50) * 2   # 真: w=2, b=1

# ── 初期化 ──────────────────────────────────────────
w, b = 0.0, 0.0
alpha = 0.01   # 学習率
n_epochs = 300 # 更新回数
N = len(x)

history = []   # 損失の記録

# ── 勾配降下ループ ────────────────────────────────────
for epoch in range(n_epochs):
    # 1. 予測
    y_pred = w * x + b

    # 2. 損失の計算
    loss = np.mean((y - y_pred) ** 2)
    history.append(loss)

    # 3. 勾配の計算（偏微分）
    dw = -2 / N * np.sum(x * (y - y_pred))
    db = -2 / N * np.sum(y - y_pred)

    # 4. パラメータの更新
    w = w - alpha * dw
    b = b - alpha * db

    if epoch % 50 == 0:
        print(f"Epoch {epoch:3d} | loss={loss:.4f} | w={w:.4f} | b={b:.4f}")

print(f"\n最終結果: w={w:.4f}, b={b:.4f}  （真: w=2.0, b=1.0）")

# ── 損失曲線のプロット ────────────────────────────────
plt.figure(figsize=(7, 3))
plt.plot(history, color="#6366f1")
plt.xlabel("Epoch")
plt.ylabel("MSE Loss")
plt.title("損失曲線（学習の進捗）")
plt.tight_layout()
plt.savefig("loss_curve.png", dpi=120)
plt.show()
出力:
Epoch 0 | loss=109.4476 | w=0.5428 | b=0.1019
Epoch 50 | loss=10.2831 | w=1.7283 | b=0.6512
Epoch 100 | loss=4.5209 | w=1.9021 | b=0.8847
Epoch 150 | loss=4.0382 | w=1.9512 | b=0.9511
Epoch 200 | loss=3.9641 | w=1.9721 | b=0.9793
Epoch 250 | loss=3.9527 | w=1.9812 | b=0.9914

最終結果: w=1.9868, b=0.9968 （真: w=2.0, b=1.0）

エポックが進むにつれ、損失が下がり、$w$ と $b$ が真の値（2.0, 1.0）に近づいていることが確認できます。

学習率の影響

学習率 $\alpha$ 挙動
0.001収束は遅いが安定。エポック数を増やす必要がある
0.01バランスが良い（上記コードの設定）
0.1速いが損失が振動することがある
1.0 以上発散する（損失が無限大に向かう）

ハンズオンでは alpha を変えながら損失曲線の形がどう変わるかを観察します。

このセクションのまとめ

  • ① 線形回帰 は $\hat{y} = wx + b$ というパラメータ付きの関数。 学習とは $w, b$ を「データに合う値」に調整すること。
  • ② 損失関数（MSE） は「予測がどれだけ外れているか」を1つの数値で表す指標。 これを最小化することが学習のゴール。
  • ③ 勾配降下法 は「損失の傾き（勾配）を計算して、坂を下る方向にパラメータを少し動かす」 という手続きの繰り返し。
