前置き

自然言語を「数値」に変換する

これまで扱ってきた線形回帰や MNIST 分類は、入力がはじめから「数値」でした。 しかし自然言語——単語や文章——はそのままでは数学的な操作ができません。 AI が言語を扱うためのカギは、言語を数値ベクトルに変換することです。

単語埋め込み（Word Embedding）の考え方

各単語を 高次元の実数ベクトル に対応させます。 ベクトルの各次元は「その単語の意味的な特性」を暗黙的に表現します。

例えば 512 次元のベクトルなら、単語は 512 個の実数の組として表現されます。 意味が近い単語は、ベクトル空間上でも近い位置に来るように学習されます。

ベクトルになると、足し算・引き算・内積といった数学的操作が直接適用できます。

ベクトル演算で意味の類推ができる

$$\vec{v}(\text{王様}) - \vec{v}(\text{男性}) + \vec{v}(\text{女性}) \approx \vec{v}(\text{女王})$$

これは「王様ベクトルから男性的な方向を引き、女性的な方向を加えると、女王ベクトルに近づく」ことを意味します。 意味の計算が、ベクトルの足し算・引き算として実現できるのです。 これはベクトル化によってはじめて可能になる操作です。

テキストが数値になるまでの流れ

テキスト
「猫が好き」
→
トークン化
「猫」「が」「好き」
→
トークンID
[2847, 14, 391]
→
埋め込み
[0.12, −0.84, …]×3

埋め込み（Embedding）は 大きな行列の参照 です。語彙サイズ × 次元数 の行列を学習し、 トークンIDを行インデックスとして対応するベクトルを取り出します。

なぜベクトル化が重要なのか

ニューラルネットワークの演算はすべて「行列の掛け算＋活性化関数」です。 テキストを実数ベクトルに変換することで、言語処理が NN の標準的な演算パイプラインに乗るようになります。 Transformer はこのベクトル列を入力として受け取り、各ベクトルを文脈に応じて変換していく構造です。

1

Section 1

Transformer 以前の問題：RNN の限界

埋め込みでテキストをベクトル列にしても、次の課題が残ります—— どうやって文脈（前後の関係）を捉えるか。 Transformer 以前は RNN（再帰型NN）がその役割を担っていましたが、根本的な限界がありました。

RNN の処理イメージ

問題① 長距離依存の忘れ

入力を逐次処理するため、文の先頭の情報は末尾に到達するまでに薄れます（勾配消失）。 「私は…（長い文）…食べた」の「私」と「食べた」の関係を捉えにくい。

問題② 並列化できない

$h_t$ は $h_{t-1}$ に依存するため、各ステップは順番に計算しなければなりません。 GPU の並列演算能力を活かせず、長い文では処理が遅くなります。

Transformer が解決したこと

Transformer は RNN を使わず、「注意機構（Attention）」だけで文脈を捉えます。 全トークン間の関係を一度に計算するため、①長距離依存を直接扱え、②並列計算が可能になります。

2

Section 2

自己注意機構（Self-Attention）

Self-Attention は「あるトークンが、文中の他のどのトークンと強く関連しているか」を数値で計算する仕組みです。 「銀行に行く」の「銀行」は金融機関か川岸か——文脈によって意味が変わります。 Self-Attention はこの文脈依存の意味を埋め込みに反映させます。

Query・Key・Value の直感

各トークンのベクトルから、3種類の異なるベクトルを生成します。

記号 名前 役割のたとえ
Q Query（質問） 「私はどの単語と関連しているか？」という問いかけ
K Key（検索キー） 「私はこんな内容を持っている」という名札
V Value（値） 関連度に応じて取り出される実際の内容

Query と Key の内積がそのトークン間の「関連度スコア」になります。 スコアを softmax で確率分布に変換し、Value の加重和を取ることで 文脈を反映した新しい表現ベクトルが得られます。

「食べた」トークンが他のトークンに注目するようす

「食べた」は「寿司（0.45）」と「昨日（0.28）」に強く注目し、「は（0.10）」にはあまり注目しません。 この重みで Value を加重平均することで、「何を・いつ食べたか」という文脈が埋め込まれた新しいベクトルが得られます。

Attention の計算式

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$
$Q, K, V$ 入力行列 $X$ に学習済み重み行列 $W^Q, W^K, W^V$ を掛けて生成
$QK^\top$ 全トークン間の「関連度スコア」行列（$n \times n$）
$\sqrt{d_k}$ 次元数の平方根でスコアをスケーリング（勾配安定化のため）
softmax スコアを確率分布（合計1）に変換
$(\cdots)V$ 確率重みで Value を加重平均 → 文脈を反映した新しい表現

Multi-Head Attention

実際には Self-Attention を複数の「ヘッド」で並列に実行します（例: 8ヘッド）。 各ヘッドは異なる $W^Q, W^K, W^V$ を持ち、異なる「注目パターン」を同時に学習できます。 一方のヘッドが「構文的な依存」を、別のヘッドが「意味的な類似性」を捉えるといった役割分担が生まれます。

Section 3

Transformer ブロックの全体構造

Transformer は Self-Attention を中核とした「ブロック」を積み重ねた構造です。 各ブロックは同じ構造を持ち、LLM ではこれを数十〜百層以上重ねます。

Transformer ブロックの内部

残差接続（Residual Connection）

入力をそのまま出力に足します（$x + \text{sublayer}(x)$）。 層が深くなっても勾配が消えにくくなり、安定した学習を可能にします。

位置エンコーディング

Self-Attention は順序を考慮しないため、トークンの位置情報を埋め込みに加算して注入します。 「1番目の単語」「2番目の単語」という位置関係を表現します。

Layer Normalization

各層の出力を正規化し、学習を安定させます。 バッチ正規化と異なり、各トークンの特徴次元方向で正規化するため、可変長の入力でも動作します。

Feed-Forward Network（FFN）

2層の線形変換（中間次元は通常 $4 \times d_{model}$）を各トークンに独立して適用します。 Attention が「どのトークンに注目するか」を決める一方、FFN が「何を記憶・変換するか」を担います。

Section 4

LLM へのつながり

Transformer は 2017 年に論文「Attention Is All You Need」で提案されました。 その後、このアーキテクチャをどう使うかによって異なる LLM が生まれました。

Transformer の使われ方

Encoder-only

BERT

入力テキスト全体を双方向で読み込み、文の意味表現を出力。 感情分析・質問応答・固有表現抽出などの分類タスクに強い。

入力 → [Encoder × N] → 意味ベクトル

Decoder-only

GPT / LLaMA / Claude

左から右へ1トークンずつ予測するテキスト生成モデル。 Causal Attention で未来のトークンを参照しないよう制限。

入力 → [Decoder × N] → 次のトークン確率

LLM（Decoder-only）が文章を生成する仕組み

$$P(\text{次のトークン} \mid \text{これまでのトークン列}) = \text{softmax}(W \cdot h_n)$$

$h_n$ は最後の Transformer ブロックが出力した最終トークンの表現ベクトル、$W$ は語彙サイズ分の出力行列です。 これを繰り返すことで、1トークンずつ文章を生成します——見かけ上「理解して答えている」ように見えますが、 本質は「次に来る確率の高いトークンを選び続ける」操作です。

なぜ LLM は賢いのか：スケーリング則

Transformer ブロックを深く・広くし、大量のテキストで次のトークンを予測し続けるだけで、 翻訳・推論・コーディングなど多様な能力が創発的に現れます。 GPT-4 は約 1 兆パラメータ規模と推定されており、MNIST で使った 20 万パラメータとは規模が 500 万倍異なります。
