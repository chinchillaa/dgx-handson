前置き

事前学習とは何か

ニューラルネットワークの学習には「正解ラベル」が必要です——線形回帰では正解値 $y$ が、 MNIST 分類では手書き数字の正解クラスが必要でした。 しかし LLM の事前学習はラベルを一切用意せず、 テキストそのものから教師信号を自動生成します。

教師あり学習（これまで）

入力 $x$ と正解ラベル $y$ のペアが必要。

画像 → ラベル「猫」「犬」
文章 → 感情「ポジティブ」

ラベル付けには人手が必要——大規模化にコストがかかる

自己教師あり学習（事前学習）

テキスト自体から教師信号を自動生成。

「猫が____」 → 正解「好き」
「今日の____は晴れ」 → 正解「天気」

インターネット上のテキストすべてが学習データになる

重要な視点

事前学習は「テキストの統計的なパターンを学ぶ」作業ですが、 大量・多様なテキストを学ぶ過程で推論・翻訳・コーディングといった能力が創発的に現れます。 これが LLM の能力の源泉です。

Section 1

次トークン予測（Causal Language Modeling）

GPT 系モデル（Decoder-only Transformer）の事前学習タスクは極めてシンプルです—— それまでのトークン列を与え、次に来るトークンを予測する。 1つの文章から複数の「問題」が自動的に生成されます。

1文から複数の学習サンプルを自動生成

「 猫  が  窓  の  外  を  眺め  た 」
入力（コンテキスト） → 正解（次トークン）
「猫」 → 「が」
「猫」「が」 → 「窓」
「猫」「が」「窓」 → 「の」
「猫」「が」「窓」「の」 → 「外」
「猫」…「外」「を」「眺め」 → 「た」

8トークンの文から7個の学習サンプルが自動生成される。 1兆トークンのデータセットなら、約1兆個の学習サンプルが得られる。

事前学習の損失関数

$$\mathcal{L} = -\frac{1}{T}\sum_{t=1}^{T} \log P(x_t \mid x_1, x_2, \ldots, x_{t-1};\, \theta)$$
$x_t$ 位置 $t$ の正解トークン
$x_1 \ldots x_{t-1}$ それまでのトークン列（コンテキスト）
$P(\cdot \mid \cdot;\,\theta)$ モデルが予測した確率分布（語彙サイズ分の softmax 出力）
$-\log P$ 交差エントロピー損失——正解トークンの確率を最大化する方向にパラメータを更新

MNIST の分類で使った CrossEntropyLoss と本質的に同じです。 クラス数が「10クラス」ではなく「語彙サイズ（数万〜数十万）」になっただけです。

Causal Masking（因果マスキング）

Decoder-only モデルでは Self-Attention の計算時に「未来のトークンを参照しない」よう 上三角行列でマスクします。これにより 1 回の順伝播で全ポジションの損失を同時に計算でき、 学習効率が大幅に向上します。

学習データ——何を、どれだけ学ぶか

事前学習の能力はデータの量・質・多様性に大きく依存します。 現代の LLM は数千億〜数兆トークンのテキストで学習されています。

データ品質は量より重要

Meta の研究（LLaMA 3）では、低品質データを除外することで同じ計算コストで大幅な性能向上が 得られることが報告されています。10兆トークンの粗いデータより、 1兆トークンの高品質データの方が優れたモデルを生む場合があります。

Section 3

スケーリング則——大きくするほど賢くなる

2020年にOpenAIが発見した「スケーリング則」は、LLM 開発の指針となっています—— パラメータ数・学習トークン数・計算量を増やすと、損失が予測可能に低下する。

べき乗則（Power Law）

$$L(N, D) \approx \frac{A}{N^\alpha} + \frac{B}{D^\beta} + L_\infty$$
$N$ モデルのパラメータ数
$D$ 学習トークン数
$\alpha,\,\beta$ スケーリング指数（実験的に決定）
$L_\infty$ データ自体のノイズによる理論的最小損失

$N$ と $D$ をそれぞれ独立に増やすと損失が下がりますが、 一方だけ増やしても効率が悪くなります。

Chinchilla 法則（2022年、DeepMind）

固定した計算予算（FLOPs）のもとで損失を最小化するには、 パラメータ数 $N$ とトークン数 $D$ の比率を約 1:20 に保つのが最適と判明しました。

Chinchilla の発見以降、LLaMA・Mistral などの効率的なモデルが続々登場しました。 現在の主流は「小さいパラメータ＋大量データ」で、推論コストを下げながら高性能を実現しています。

Section 4

事前学習済みモデルの活用方法

事前学習は莫大な計算資源を要します。 ほとんどの実務では事前学習済みモデルを出発点として活用します。 目的・予算・性能要件によって4つのアプローチが使い分けられます。

① ゼロショット
追加学習なし

事前学習済みモデルをそのままプロンプトで指示する。 「このメールを要約してください」のように自然言語で指定するだけ。

適用場面: 汎用タスク・プロトタイピング。追加コストなし。

② Few-shot プロンプティング
追加学習なし

プロンプトに入出力の例を数件（2〜10件）示すことでタスクを定義する。 モデルはパターンを読み取りタスクを遂行する。

適用場面: 少数例で形式を揃えたいとき。コンテキスト長の消費に注意。

③ ファインチューニング（SFT / LoRA）
追加学習あり

タスク固有データでパラメータを更新し、特定ドメインへの適応・指示への応答能力を強化する。 LoRA（Low-Rank Adaptation）を使えば、更新するパラメータを全体の 1% 以下に絞れる。

適用場面: 社内文書・専門用語・特定スタイルへの適応。第2章のメインテーマ。

④ RAG（検索拡張生成）
追加学習なし

質問に関連する文書をベクトル検索で取得し、コンテキストとしてプロンプトに挿入する。 モデルのパラメータを変えずに「最新情報」「社内知識」を参照させられる。

適用場面: 知識ベースの頻繁な更新・ハルシネーション抑制。第2章のメインテーマ。
