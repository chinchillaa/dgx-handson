前置き

線形回帰では解けない問題がある

前のセクションで「AI は関数である」という視点を得ました。 しかし、線形回帰（$\hat{y} = \mathbf{W}\mathbf{x} + \mathbf{b}$）は 直線・平面でしか境界を引けない という根本的な限界があります。 現実の問題の多くはこの限界を超えています。

例：XOR 問題（線形分離不可能）

XOR は「2つの入力が異なるとき 1、同じとき 0」を返す演算です。 4点を散布図に描くと、1本の直線で ○ と × を分けることは不可能 です。

$x_1$ $x_2$ $y$（正解）
000 ×
011 ○
101 ○
110 ×

なぜ線形変換では解けないのか？

線形変換をいくつ重ねても、結果は常に「1回の線形変換」と数学的に等価です。 つまり、どれだけ層を増やしても 曲線的な境界を表現できません。 この問題を解決するのが 活性化関数（非線形変換） の導入です。

Section 1

ニューラルネットワークの構造

ニューラルネットワーク（NN）は「関数を複雑にするための部品」を積み重ねた構造です。 線形変換 → 活性化関数 → 線形変換 → … というブロックを繰り返すことで、任意の複雑な入出力関係を近似できます。

2層ニューラルネットワークの構造

入力層（3ノード）→ 隠れ層（3ノード、活性化関数あり）→ 出力層（2ノード）

各部品の役割

ノード 1つの数値を保持する計算単位。「ニューロン」とも呼ぶ
$\mathbf{W}$ 重み行列。前の層の出力を次の層の入力に変換するパラメータ
$\mathbf{b}$ バイアスベクトル。変換後の値を平行移動させるパラメータ
$f(\cdot)$ 活性化関数。線形変換の後に適用し、非線形性を加える
層（layer） 「線形変換 + 活性化関数」のセット。これを積み重ねる
2

Section 2

活性化関数：「非線形にする部品」

活性化関数は各ノードの出力に適用される非線形変換です。 これがなければ、何層重ねても1回の線形変換と等価になってしまいます。 代表的な2つを押さえておきましょう。

ReLU

$$ f(x) = \max(0, x) $$

負の値をすべて 0 に、正の値はそのまま通す。 現代の NN で最も広く使われる 活性化関数。

計算が軽く、勾配消失が起きにくい。深い NN に向いている。

Sigmoid

$$ f(x) = \frac{1}{1 + e^{-x}} $$

任意の実数を 0〜1 の範囲に押し込む。 出力を「確率」として解釈したい場合（2値分類の出力層）に使われる。

深い層では勾配消失が起きやすく、隠れ層では ReLU に置き換わった。

「どの活性化関数を使うか」は設計の選択です。 隠れ層はほぼ ReLU（またはその派生形）、出力層は問題に応じて選ぶのが現代の標準です。 大切なのは「非線形変換が入ることで関数が複雑な形を表現できる」という事実です。

Section 3

順伝播と PyTorch 実装

入力から出力まで順番に計算を進めていく処理を 順伝播（Forward Propagation） と呼びます。 2層NN の場合、数式で書くと次のようになります。

2層NN の順伝播

$$ \mathbf{h} = f\!\left(\mathbf{W}^{(1)}\mathbf{x} + \mathbf{b}^{(1)}\right) \quad \text{（隠れ層）} $$ $$ \hat{\mathbf{y}} = \mathbf{W}^{(2)}\mathbf{h} + \mathbf{b}^{(2)} \quad \text{（出力層）} $$
$\mathbf{x}$入力ベクトル
$\mathbf{W}^{(l)}, \mathbf{b}^{(l)}$第 $l$ 層の重み行列・バイアスベクトル（学習対象）
$f(\cdot)$活性化関数（ReLU など）
$\mathbf{h}$隠れ層の出力（次の層への入力）
コード例 — PyTorch で 2層NN を実装（nn.Module 最小構成） 
import torch
import torch.nn as nn

# 実行時間: 数秒

# ── モデルの定義 ────────────────────────────────────
class TwoLayerNN(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        super().__init__()
        # 第1層: 線形変換
        self.fc1 = nn.Linear(input_dim, hidden_dim)
        # 活性化関数
        self.relu = nn.ReLU()
        # 第2層: 線形変換
        self.fc2 = nn.Linear(hidden_dim, output_dim)

    def forward(self, x):
        # 順伝播: 入力 → 隠れ層 → 活性化 → 出力
        h = self.relu(self.fc1(x))   # h = ReLU(W¹x + b¹)
        y = self.fc2(h)              # ŷ = W²h + b²
        return y

# ── インスタンス化 ──────────────────────────────────
model = TwoLayerNN(input_dim=3, hidden_dim=8, output_dim=2)
print(model)

# ── パラメータ数の確認 ──────────────────────────────
total_params = sum(p.numel() for p in model.parameters())
print(f"\n総パラメータ数: {total_params}")

# ── 順伝播のテスト ──────────────────────────────────
x = torch.randn(1, 3)          # バッチサイズ1、入力3次元
output = model(x)
print(f"入力の形状:  {x.shape}")
print(f"出力の形状:  {output.shape}")
出力

  (fc1): Linear(in_features=3, out_features=8, bias=True)
  (relu): ReLU()
  (fc2): Linear(in_features=8, out_features=2, bias=True)
)

出力の形状: torch.Size([1, 2])

forward() メソッドに「どう計算するか」を書くだけです。 逆伝播（勾配計算）は PyTorch が自動で行います。

Section 4

自動微分（autograd）：手で微分しなくていい理由

線形回帰のセクションでは、損失の勾配を 手で計算して コードに書き下しました。しかし NN のパラメータは数百万〜数十億もあります。 それを人手で微分するのは現実的ではありません。

PyTorch の autograd は、計算グラフを自動構築して 任意のパラメータに対する勾配を自動計算します。 研究者はモデルの構造（forward()）を書くだけで、勾配計算は PyTorch が肩代わりします。

autograd が実現したこと

2012年以降の深層学習ブームの背景の一つは「自動微分フレームワークの普及」です。 研究者がモデルの構造設計に集中できるようになり、 100層を超えるネットワークの学習が現実的になりました。

コード例 — autograd による勾配の自動計算 
import torch
import torch.nn as nn

# 実行時間: 数秒

# ── データ（ダミー） ────────────────────────────────
x      = torch.randn(32, 3)   # バッチサイズ32、入力3次元
y_true = torch.randn(32, 2)   # 正解ラベル（2次元）

# ── モデル・損失関数・最適化アルゴリズム ────────────
model     = TwoLayerNN(input_dim=3, hidden_dim=8, output_dim=2)
criterion = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

# ── 1ステップの学習ループ ────────────────────────────
# 1. 順伝播
y_pred = model(x)

# 2. 損失計算
loss = criterion(y_pred, y_true)
print(f"損失: {loss.item():.4f}")

# 3. 勾配を初期化（重要：前ステップの勾配が残るため）
optimizer.zero_grad()

# 4. 逆伝播（autograd が全パラメータの勾配を自動計算）
loss.backward()

# 5. パラメータ更新
optimizer.step()

# 勾配の確認（fc1 の重みに対する勾配）
print(f"fc1.weight の勾配（形状）: {model.fc1.weight.grad.shape}")
print(f"勾配の例: {model.fc1.weight.grad[0, :].detach().numpy().round(4)}")
出力
損失: 1.0823
fc1.weight の勾配（形状）: torch.Size([8, 3])
勾配の例: [ 0.0031 -0.0147 0.0089]

loss.backward() の1行で、全パラメータの勾配が自動計算されます。 線形回帰で手書きした偏微分式を、ここでは一切書いていません。

PyTorch 学習ループの5ステップ

  1. y_pred = model(x) — 順伝播で予測を計算
  2. loss = criterion(y_pred, y_true) — 損失を計算
  3. optimizer.zero_grad() — 前ステップの勾配をリセット
  4. loss.backward() — 逆伝播（autograd が勾配を自動計算）
  5. optimizer.step() — 勾配に従いパラメータを更新

この5ステップは PyTorch の学習ループの黄金パターンです。 MNIST 分類でも LLM のファインチューニングでも、本質的にはこの繰り返しです。

このセクションのまとめ

  1. 線形回帰の限界：直線で分離できない問題（XOR など）は解けない。 「非線形な関数」が必要。
  2. ニューラルネットワークは「線形変換 + 活性化関数」のブロックを積み重ねた構造。 層を深くするほど複雑な入出力関係を表現できる。
  3. 活性化関数（ReLU など）が非線形性を加える。これがなければ何層重ねても意味がない。
  4. autograd により勾配は自動計算される。 forward() を書けば PyTorch が逆伝播を肩代わりする。
